# Recura Implementation Plan

## Project shape
- Independent repo from PoliMetrics
- One-page German business website
- Very close adaptation of the supplied mockups
- Stack: Next.js + Tailwind CSS

## Locked decisions
- Navigation: Leistungen, Use Cases, Über uns, Kontakt
- Contact flow: booking CTA + email option + contact form
- Team names are treated as real content
- Placeholder values are acceptable for booking link and public email in the first version

## Core sections
1. Hero
2. Leistungen / Use Cases
3. Über uns
4. Unser Vorgehen
5. Kontakt
6. Footer
7. Separate pages: Impressum, Datenschutz

## Implementation phases
### Phase 1
- Scaffold app and repo foundation
- Add brand palette, typography, assets, layout primitives
- Create project TODO tracking

### Phase 2
- Build homepage sections in production layout order
- Match mockup spacing, card styles, and CTA structure closely

### Phase 3
- Add legal pages
- Responsive refinement
- SEO and performance pass
- Form and scheduling placeholders clearly marked

## Design notes
- Clean B2B look, white/soft background, teal + navy accents
- Close visual adaptation, not a generic startup template
- Trust, clarity, and pragmatic automation should come through the copy and layout
